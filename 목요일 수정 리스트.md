# 목요일 수정 리스트

## 드론 원점 진동/오버슈트 원인 및 수정 항목

### 1. PSC(위치·속도 컨트롤러) 게인 과다 — 가장 유력

| 파라미터 | 기본값 | 권장값 | 영향 |
|---|---|---|---|
| `PSC_POSXY_P` | 1.0 | 0.5 | 위치 오차 → 속도 명령 변환 배율. 높으면 조금만 멀어져도 과속 명령 |
| `PSC_VELXY_P` | 2.0 | 1.0 | 속도 오차 → 기울기 명령. 높으면 원점 지나쳐 반대편으로 돌진 |
| `PSC_VELXY_I` | 1.0 | 0.5 | 적분항. 오차 누적 시 오버슈트 |
| `PSC_VELXY_D` | - | 0.25 | 미분항. 낮추면 진동 감소 |
| `LOIT_SPEED` | 500cm/s | 낮춤 | 위치 제어 최대 수평 속도. 높으면 접근 속도 자체가 빠름 |

### 2. VPE 전송 지연(latency) → 제어 위상 지연

- UWB 측정 → 파싱 → `_vision_loop` 전송 → EKF 융합 → 컨트롤러 반영까지 수십~수백ms 지연
- 컨트롤러가 "지금 위치"가 아닌 "과거 위치" 기반으로 명령 → 실제 드론은 이미 이동 완료인데 계속 가속 명령 → 오버슈트

### 3. VPE covariance 너무 타이트

- `lib_common.py`: `cov[0] = cov[6] = 0.01` (xy ±10cm)
- EKF이 UWB 값을 거의 100% 신뢰 → UWB에 멀티패스 노이즈·위치 점프 섞이면 EKF 출력이 그대로 튀고 컨트롤러 과반응
- 값을 키워서 EKF가 IMU와 융합하도록 조정 검토

### 4. go_to() 전송 주기 2Hz — 위치 목표 갱신 느림

- `flight.py` hover loop: `time.sleep(0.5)` → 0.5초마다 `SET_POSITION_TARGET` 전송
- 컨트롤러는 이전 목표를 0.5초 유지하다 갑자기 새 목표로 전환 → 불연속 제어
- 10Hz 이상으로 높이는 것 검토

### 5. UWB 실내 멀티패스 노이즈

- 실내 벽·금속 구조물에서 UWB 신호 반사(multipath) → 위치값 수십cm 단위 점프
- EKF가 점프를 위치 이동으로 해석 → 컨트롤러 과반응 → 진동
- 앵커 배치 조정 또는 소프트웨어 필터 추가 검토

### 6. ATC(자세 컨트롤러) 기체 PID 미튜닝

- `ATC_RAT_RLL_P/D`, `ATC_RAT_PIT_P/D` 기본값이 기체 무게/모터 응답과 맞지 않으면 자세 제어 자체가 오버슈트 → 위치 진동 증폭
- VPE 기반 실내에서 `ATC_RAT_RLL_D=0.0001`로 낮추는 사례 있음 (ArduPilot Discourse)

### 7. EKF 위치 추정 노이즈

- VPE만으로 xy 추정 시 IMU 드리프트 + UWB 노이즈가 섞인 EKF 출력이 0.1~0.2m 단위로 흔들림
- 컨트롤러 입장에서 드론이 "계속 움직이는 것처럼" 보임 → 제자리인데 계속 보정 명령 → 진동

---

**수정 우선순위:** PSC 게인 과다 > VPE latency > go_to() 2Hz 갱신 > UWB 멀티패스
